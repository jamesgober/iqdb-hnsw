# iqdb-hnsw v1.0.0 — Stable

**The production ANN index is stable.** `iqdb-hnsw` graduates to `1.0` and joins
the iQDB family's stable line. The public surface frozen in `0.6.0` is unchanged
and is now committed until `2.0`; this release closes the last validation gap —
recall is now measured against the published `iqdb-flat` ground truth — and
polishes the internals. There are **no breaking changes** from `0.6.0`.

## What is iqdb-hnsw?

The production approximate-nearest-neighbour layer of the iQDB vector database — a
Hierarchical Navigable Small World graph (Malkov–Yashunin, 2016) that answers
top-`k` queries by beam-search descent through a multi-layer proximity graph.
After the types crate it is the most important crate in the family: it is what
makes vector search over millions of vectors fast. It implements the
`iqdb-index` `Index` trait, so it is interchangeable with the exact `iqdb-flat`
and the clustered `iqdb-ivf` behind one interface.

## Why 1.0 now

The crate has been feature-complete and API-frozen since `0.6.0`. Its entire
dependency closure — `iqdb-types`, `iqdb-distance`, `iqdb-index`, `iqdb-filter` —
plus its recall-oracle sibling `iqdb-flat` are all published and stable at `1.0`.
The one alpha task that actually gated the public surface was recall validation
against `iqdb-flat` rather than a hand-rolled scan; with `iqdb-flat 1.0` on
crates.io that is now done. The remaining roadmap items (an hnswlib external
comparison, broader real-data sweeps) are out-of-tree diagnostics, not API gates,
so they do not block the stable tag.

## What changed since 0.6.0

### Recall validated against `iqdb-flat`

The headline recall gate (`tests/recall.rs`) and the documentation bench
(`benches/hnsw_bench.rs`) now score HNSW against `iqdb_flat::FlatIndex`'s exact
top-`k`, not an inline full-scan oracle. `FlatIndex` carries the same ordering
contract (`Hit.distance` smaller-is-nearer, `DotProduct` negated at the boundary)
and the same insertion-order tie-break, so the comparison is apples-to-apples and
the gate measures the *graph*. This fulfils the DIRECTIVES §8 mandate that recall
be measured against `iqdb-flat`. The 0.95 recall@10 floor across all five metrics
is unchanged and still met.

### Filtered-traversal coverage

A new `tests/filter.rs` suite exercises the `SearchParams::filter` path end to
end: matching-only results, an empty result on a non-matching predicate,
exclusion of records that carry no metadata, and correct interaction with
tombstones. The worked filtered-search example in `docs/API.md` is the same code,
kept honest as a compiled test.

### Internal cleanup

- Removed the unused `select_simple` neighbour-selection helper (a dead
  diagnostic) and the unused `query` parameter on the internal `select_heuristic`
  — both private, no API impact.
- Refreshed the README, `docs/API.md`, and the ROADMAP to the stable status;
  removed stale pre-1.0 references.

## The surface (unchanged, now frozen until 2.0)

- `VERSION`, the `Hit` re-export, `HnswConfig`, and `HnswIndex`.
- `HnswConfig` — `m`, `ef_construction`, `ef_search`, `filter_widen`, `seed`,
  with builder-style `with_*` overrides and documented defaults (`m = 16`,
  `ef_construction = 200`, `ef_search = 64`, `filter_widen = 4`).
- `HnswIndex` — `impl Index` (`type Config = HnswConfig`) and the full
  `IndexCore` implementation (insert, delete, batch insert, single and batch
  search, `flush`, `stats`), plus inherent `new_unconfigured`, `dim`, `metric`,
  `len`, `is_empty`, `config`, and a `node_layer_histogram` diagnostic.
- No feature flags.

Frozen behavioural contracts: smaller-is-nearer ordering across all five metrics
(`DotProduct` negated); deterministic graph under a fixed seed + insert order;
tombstone delete (a deleted id never reappears until re-inserted); no panics on
any input; zero `unsafe`. `iqdb_types::IqdbError` is `#[non_exhaustive]`, so new
error variants remain non-breaking. Only **additive, non-breaking** changes are
made through the 1.x series.

## Dependencies

Wired to the stable iQDB spine — all `1.0`:

```toml
iqdb-types    = "1.0.0"
iqdb-distance = "1.0.0"
iqdb-index    = "1.0.0"
iqdb-filter   = "1.0.0"
```

`iqdb-flat 1.0` is a **dev-dependency only** — the recall oracle — and is not part
of the runtime closure. There are no feature flags.

## Breaking changes

**None.** The public API is identical to `0.6.0`; only private internals and tests
changed.

## Verification

Clean across the CI matrix (Linux, macOS, Windows) on stable and the 1.87 MSRV:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo deny check
```

Zero `unsafe` (`#![forbid(unsafe_code)]`); no `unwrap`/`expect`/`todo!`/`dbg!` in
shipping code. The recall gate passes against the exact `iqdb-flat` ground truth
across all five metrics; the SIFT-1M diagnostic (`tests/sift_recall.rs`) is
available for real-data validation when the dataset is present.

## What's next

- **Slot compaction / graph-repair delete** remains a tracked post-`1.0` internal
  optimisation (see the ROADMAP note). Delete is tombstone-only until then; the
  observable contract holds and it does not change the public API.
- **Consumer integration** continues through the `iqdb` engine and `iqdb-build`.

## Installation

```toml
[dependencies]
iqdb-hnsw = "1.0"
```
